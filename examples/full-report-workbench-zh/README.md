# 模拟经营复盘报告写作工作台

这是一个脱敏、合成的 full example，用来展示 Writing Workbench 如何承载一个多素材、多章节、多轮迭代的报告写作项目。

场景设定：团队 Alpha 完成了 8 个阶段的模拟经营，需要写一份复盘报告。报告要解释经营结果、阶段性决策、方法改进和后续建议。所有数字、团队名称和素材都是合成数据。

## 先读这些

1. `00_Context/00_项目简报.md`
2. `00_Context/01_数据口径与核心事实.md`
3. `00_Context/02_文风与禁忌.md`
4. `00_Context/03_素材归类与对账.md`
5. `01_Workflow/结构方案.md`
6. `01_Workflow/写作流程.md`
7. `01_Workflow/章节写作看板.md`
8. `04_Drafts/模拟经营复盘报告_v1.md`
9. `05_Checklists/工作台基座验收.md`（阶段性检查工作台本身）

## 目录职责

| 目录 | 用途 |
| --- | --- |
| `00_Context/` | 每次开工都要读的稳定口径，带来源索引 |
| `01_Workflow/` | 结构方案、实际写作顺序、依赖关系和单模块调用方式 |
| `02_Modules/` | 每章任务书 |
| `03_DataRoom/` | 素材索引和合成数据入口 |
| `04_Drafts/` | 草稿和后续合稿 |
| `05_Checklists/` | 模块验收与全篇验收 |

如果只想理解骨架，先读六层目录下的 `README.md`。它们说明每一层的职责、加载时机和边界。

本例展示了两个显式化机制：`00_Context/` 中的压缩口径都有来源索引；`01_Workflow/结构方案.md` 区分读者看到的成稿结构和作者实际推进的写作顺序。

## 当前口径

- 报告范围是阶段 1-8。
- 前 4 个阶段主要依赖人工判断。
- 后 4 个阶段引入结构化检查表辅助决策。
- 合成指标表是报告中的唯一数字来源。
- 报告重点是复盘经营节奏和方法迭代，不写成工具实现说明。

## 下一步

如果继续写作，先看 `01_Workflow/结构方案.md`，再按 `01_Workflow/章节写作看板.md` 的实际写作顺序逐章推进。每章开工前加载全部 `00_Context/`，再加载当前模块任务书和 DataRoom 中点名素材。写完后保存到 `04_Drafts/`，并对照 `05_Checklists/` 验收。
