# 00_Context — 预加载上下文层

本层存放每次写作前都要加载的小而稳定的信息。它回答：这份报告写给谁、写什么、用什么数字口径、按什么文风写。

本例把 `00_Context` 当作带来源索引的压缩上下文层：正文写作时加载压缩后的稳定口径；需要复查时，沿每份文件里的 `## 来源索引` 回到 Inventory ID 和示例内来源位置。

## 本例文件

- `00_项目简报.md`：报告定位、读者、交付边界。
- `01_数据口径与核心事实.md`：阶段范围、合成数据来源、核心事实。
- `02_文风与禁忌.md`：写作风格、推荐写法和禁止写法。
- `03_素材归类与对账.md`：Inventory 分区总账。

## 使用规则

写任何模块前先读本层全部文件。发现跨章节共享规则变化时，先改本层，再改草稿。新增或修改 Context 内容时，同步更新对应文件的 `## 来源索引`。
