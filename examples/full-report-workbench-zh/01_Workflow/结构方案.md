# 结构方案

本文件记录 Step 5 的结构判断。它把作者初步想法、目标读者偏好和领域惯例显式摆出来，再决定成稿结构和实际写作顺序。

## 判断输入

| 输入 | 当前判断 |
| --- | --- |
| 作者初步想法 | 想写一份能解释阶段 1-8 经营变化的复盘报告，但不把它写成流水账。 |
| 目标读者偏好 | 教练和接手者需要先看结果边界，再看过程解释，最后看到可复用方法。 |
| 领域惯例 | 经营复盘通常按“结果 - 过程 - 方法 - 下一步”组织，但写作时应先把数据和过程写稳。 |

这三类输入都不是自动正确答案。最终结构要服务于本例的主线：从直觉经营到结构化复盘。

## 候选方案

| 方案 | 成稿结构 | 实际写作顺序 | 优点 | 风险 |
| --- | --- | --- | --- | --- |
| A. 结果先行 | 摘要与边界 → 总体表现 → 阶段复盘 → 方法迭代 → 结论与下一步 | 总体表现 → 阶段复盘 → 方法迭代 → 摘要与边界 → 结论与下一步 | 读者先知道范围和结果，写作时先稳定数字和过程。 | 摘要如果过早写，容易重复正文数字。 |
| B. 过程先行 | 摘要与边界 → 阶段复盘 → 总体表现 → 方法迭代 → 结论与下一步 | 阶段复盘 → 总体表现 → 方法迭代 → 摘要与边界 → 结论与下一步 | 适合强调团队如何逐步调整。 | 读者要更晚看到总结果，报告开头的判断力度较弱。 |
| C. 方法先行 | 摘要与边界 → 方法迭代 → 总体表现 → 阶段复盘 → 结论与下一步 | 方法迭代 → 总体表现 → 阶段复盘 → 摘要与边界 → 结论与下一步 | 把检查表作为全文主线，方法感强。 | 容易让读者误以为报告重点是工具，而不是经营复盘。 |

## 最终选择

采用方案 A。

成稿结构采用“结果先行”，因为读者需要先确认报告范围、数据口径和总体改善，再进入阶段过程。实际写作顺序从“总体表现”开始，因为数字是全文锚点；再写“阶段复盘”和“方法迭代”；最后回写“摘要与边界”和“结论与下一步”，避免开头和结尾在事实尚未稳定时先行定调。
