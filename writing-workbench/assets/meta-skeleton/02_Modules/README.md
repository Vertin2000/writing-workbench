# 02_Modules — 模块任务书层

## 角色

每份文件是**一次 AI 调用就能写完的最小独立单元**的任务书。不论这一单元在你的项目里叫章 / 节 / 部分 / 幕 / 篇 / 标段——本层只关心：它是一次写作迭代的目标。

任务书不是草稿，也不是大纲。它是写这块时给 AI 的"工作命令"。

## 何时加载 / 何时写入

- **写入时机**：Modules 切分定下后，为每个模块各建一份任务书。任务书内容由用户在 Step 7 (Fill) 填入。
- **加载时机**：当前要写哪个模块，就只加载哪个模块的任务书。其他模块的任务书不加载（避免 AI 串戏）。
- **更新时机**：写到一半发现交付物 / 边界判断需要调整，回到任务书改完再继续；不要在草稿里偷偷漂移。

每份任务书文件以 `# 模块名` 为文件标题，五要素各用一个 `##` 二级标题。

## 五要素（每份任务书都应包含）

1. **定位**：这一块要回答什么问题、对全篇起什么作用、面向谁、读完应该带走什么
2. **必读材料**：本模块写作前要加载的具体素材清单（指向 03_DataRoom 文件、其他模块草稿、外部参考等）
3. **最小交付物**：本模块写完之后产出哪些段落 / 表格 / 图 / 结构件，缺一不可
4. **不得写入内容**：哪些话题留给其他模块，本模块不要碰（防越界，防重复）
5. **验收清单**：本模块独立验收的逐项检查。**05_Checklists 是权威源，本字段仅以路径引用 05_Checklists 中的对应清单文件**，不重复内容。

**不预设小节级结构**。任务书不规定"先写三段再写一表"，那是写作时由 AI 决定的输出形态。

## 与其他层的关系

- 任务书引用的全篇规范 → 在 00_Context 已加载，不必重复
- 任务书要用的大体积素材 → 不内嵌，仅在"必读材料"中标注 03_DataRoom 文件路径
- 草稿产出 → 04_Drafts，不写在任务书里
- 验收 → 单向引用 05_Checklists 中的对应清单文件（权威源在 05_Checklists）

## 完整性约束

每个 Step 5 切出来的模块**都必须有任务书**。不允许"这块不重要先不写任务书"——没有任务书就不要开始写正文，否则 AI 没有边界判断标准，会写得越界或重复。
