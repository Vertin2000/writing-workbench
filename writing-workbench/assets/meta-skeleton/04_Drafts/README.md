# 04_Drafts — 草稿层

## 角色

模块写作的**产出物**寄存处。每个模块的草稿、修订版本、合稿版本都放这里。区别于 02_Modules（任务书 = 写之前），本层是"写之后"。

## 何时加载 / 何时写入

- **写入时机**：单模块写作完成时立即保存到本层。文件名建议形如 `<模块名>_draft_v<n>.md`，每次显著修订递增版本号。
- **加载时机**：
  - 写当前模块时一般不加载本层任何文件
  - 写下游模块（依赖上游模块结论）时，按需加载上游模块的最新草稿
  - 验收阶段、合稿阶段加载相关草稿
- **更新时机**：每次修改另存为新版本，老版本保留以备回溯（除非确认无用）。

## 与其他层的关系

- 本层**只放草稿**，不放任务书、不放素材、不放规范
- 草稿引用的素材路径仍指向 03_DataRoom 与 00_Context，不在本层重复
- 验收发现问题时，先回 02_Modules 任务书 / 00_Context 确认是不是规则不清，再决定改草稿还是改规则

## 编译 / 导出

最终合稿与导出（Word / PDF / LaTeX 等）由其他 Skill 接手（如 typora-pandoc-zotero-thesis、docx）。本层只保留 Markdown 源稿。

## 完整性约束

每个模块至少应有一份对应草稿（一旦该模块开始写）。半成品也要落盘成 vN 草稿，不要把模块草稿留在对话里——对话会丢，草稿不丢。
