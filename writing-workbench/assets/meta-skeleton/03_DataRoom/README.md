# 03_DataRoom — 外部素材层

## 角色

体积大、按需加载、不进 Context、不进任务书的**外部素材索引与寄存处**。它是项目的"档案室"：AI 不默认读，由具体模块任务书在"必读材料"里点名调用。

## 何时加载 / 何时写入

- **写入时机**：Inventory 阶段把所有外部素材的清单整理好，本层为每类素材建一份索引文件（或一个子目录）。素材本体可以原地放在外部目录、本层只放索引；也可以本层就是素材本体的家。
- **加载时机**：仅在某个模块的任务书"必读材料"显式列出某项素材时才加载。**不**在每次开工时全量加载。
- **更新时机**：补充新素材、淘汰旧素材时同步索引。

## 本层可能放什么（按项目自行决定）

- 原始数据表（Excel / CSV / 数据库导出）
- 参考资料文献（PDF 研报 / 论文 / 法规原文 / 判例 / 经典文本）
- 图、谱系图、地图、设计稿
- 角色档案 / 世界观设定文档 / 时间线
- 历史草稿 / 既有版本
- 外部访谈、笔记
- 翻译参考文本对照
- 生成型素材：脚本、CSV/Markdown 源表、PNG/SVG 图表、可视化资产、生成命令
- 任何"想留着备查但不想每次都读"的东西

## 与其他层的关系

- 跨模块都共享、又**小**的素材性内容（如术语表）→ 不在本层，在 00_Context
- 大体积、只在某些模块要用的素材 → 本层
- 需要现场加工的素材（如把数据生成图表）→ 本层存原始数据，加工脚本与中间产物视情况
- 由脚本生成的源表和图表 → 留下原始来源、生成命令和口径说明，方便复查
- 按需 handoff 包复制了图表、截图、数据源等素材 → 本层仍是追溯源，handoff 的 README 需写明来源路径

## 可全删

如果项目**没有任何**外部素材（纯创作类、纯思辨类），本层可整个删除，不影响其他五层骨架运转。

**时序提示**：Scaffold 阶段默认保留本层（即使暂时没素材），等项目走完一轮、确认整篇都不会用到外部素材后再清理 — 比一开始就删更稳。

## 完整性约束

凡 Partition 阶段被分到本层的素材，**必须在某个模块任务书的"必读材料"里被引用**——否则它就是无人使用的素材，要么删除，要么搬到 00_Context。本层不允许"放着没人用"的孤儿文件。
